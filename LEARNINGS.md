## Learnings
* Positioning elements absolute within a relative container that has SVG shapes allows for complex, visually precise overlays (e.g., placing a logo perfectly between the peaks of an SVG "M" path).
* To ensure smooth CSS transitions on dynamic pseudo-elements (`::after`), transition properties must cover `transform` and `opacity` with customized cubic-bezier timing functions (`cubic-bezier(0.4, 0, 0.2, 1)`).
* When applying transformations via CSS, layering `z-index` properly on both `:hover` and `.active` states guarantees that expanding dots and their labels won't be eclipsed by adjacent UI elements.

* Django Error pages (404, 500, etc) can be cleanly overridden by creating corresponding HTML files inside templates and linking them in the main urlconf using `handler404`, `handler500`, etc.
* To extend CKEditor functionalities such as adding external Youtube video embeds in Django, download the necessary CKEditor plugins to your static directory and configure `CKEDITOR_CONFIGS` in your settings to include `extraPlugins: 'youtube'` and add it to the toolbar.
* Error pages can easily be made standalone (without a global header/footer) by omitting `{% extends 'base.html' %}` and providing a complete `<html>` document structure within the error template files.
* Professional styling is achieved by minimizing harsh colors, relying on elegant typography combos (like `Cormorant Garamond` headers with `Inter` body text), and substituting chaotic animations with slow-breathing, low-opacity ambient glows.
* `django-ckeditor` (version 4) is deprecated. Upgrading to `django-ckeditor-5` requires changing the field types in models from `RichTextUploadingField` to `CKEditor5Field`, replacing configurations with `CKEDITOR_5_CONFIGS`, altering the URL routes, and carefully substituting any lingering old field names directly in migration files if migrating over an existing installation.
* Verifying rich text posts requires matching the correct url configuration and parsing the resulting HTML payload to confirm expected markup (e.g. `<oembed>` or `<img>`) is successfully saved and rendered.
* To create a sophisticated watermark effect on web pages, apply `font-size: 25vw`, low opacity (`rgba(255, 255, 255, 0.02)`), and position it absolutely in the center with `pointer-events: none` and `user-select: none` to avoid interfering with user interaction.
